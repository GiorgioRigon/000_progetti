  È una buona idea creare 1-2 template di prompt che ti soddisfano davvero e usarli come base per WB1. È coerente con il perimetro attuale: niente automazione fragile, ma migliore qualità dell’input e quindi risultati migliori.

  Ti suggerisco questo approccio semplice:

  1. prepara due prompt reali che useresti volentieri
  2. falli il più diversi possibile nello stile
  3. per ciascuno, conserva anche un mini commento su:
      - quando funziona bene
      - che tipo di risultati produce
      - quali limiti ha
  4. poi li facciamo leggere a WB1 per ottenere:
      - un prompt base unico più robusto
      - una versione personalizzata per singola organization
      - una base riusabile per una futura versione più automatizzata

  La cosa importante è che i template non siano generici, ma strutturati bene sugli input che contano davvero. Per WB1, gli input utili sono questi:

  - nome organization
  - tipo di organization
  - territorio
  - sito già noto, se esiste
  - canali già noti
  - obiettivo della ricerca
  - dati da trovare
  - criteri di verifica
  - formato di output desiderato

  Quindi sì: il prossimo micro-passo sensato è che tu prepari 1 o 2 prompt template “buoni”, e al turno dopo lavoriamo solo su quello. Possiamo:

  - ripulirli
  - confrontarli
  - estrarne una struttura comune
  - trasformarla nel prompt di WB1
